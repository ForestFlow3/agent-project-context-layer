# Synthetic 示例：Vibe Coding App Project / Synthetic Example: Vibe Coding App Project

## 中文

这个示例是 synthetic 的。

它展示一个独立创作者如何启动一个小型 coding-agent 辅助项目，并让项目上下文在 Deepseek、Kimi Code、Trae、Codex、Claude Code、Cursor 或其他 agent 中保持可读。

### 场景

一位创作者想构建一个 local-first 灵感收集箱，名为 `Spark Shelf`。

这个 App 帮助创作者收集原始想法、打标签，并把选中的 notes 转成内容大纲。

### 输入

- `inputs/idea-note.md` 记录初始产品想法。
- `inputs/session-note-01.md` 记录第一次范围决策。
- `inputs/feature-notes.md` 记录字段、风险和下一步行动。

### 输出

- `outputs/README.md` 展示被动 context layer 清单。
- `outputs/project-context-package.json` 展示结构化 context package。
- `outputs/PROJECT_CONTEXT.md` 展示稳定项目上下文。
- `outputs/AGENT_BRIEF.md` 展示 agent 交接 brief。
- `outputs/DECISIONS.md` 展示已接受决策。
- `outputs/RISKS.md` 展示已知风险。
- `outputs/TODO.md` 展示下一步行动。

## English

This example is synthetic.

It demonstrates how a solo creator might start a small coding-agent-assisted project and keep project context readable across Deepseek, Kimi Code, Trae, Codex, Claude Code, Cursor, or another agent.

### Scenario

A creator wants to build a local-first inspiration inbox called `Spark Shelf`.

The app helps the creator collect raw ideas, tag them, and turn selected notes into content outlines.

### Inputs

- `inputs/idea-note.md` records the initial product idea.
- `inputs/session-note-01.md` records the first scope decision.
- `inputs/feature-notes.md` records fields, risks, and next actions.

### Outputs

- `outputs/README.md` shows the passive context-layer manifest.
- `outputs/project-context-package.json` shows the structured context package.
- `outputs/PROJECT_CONTEXT.md` shows the stable project context.
- `outputs/AGENT_BRIEF.md` shows the agent handoff brief.
- `outputs/DECISIONS.md` shows accepted decisions.
- `outputs/RISKS.md` shows known risks.
- `outputs/TODO.md` shows next actions.
