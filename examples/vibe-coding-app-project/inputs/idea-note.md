## English

Source ID: S001

Source type: synthetic model-generated note

Date: 2026-06-09

I want to build a tiny local-first app called `Spark Shelf`.

The problem is that I collect ideas in too many places, then lose the context before I turn them into posts, mini tools, or project briefs.

The first version should be simple:

- Paste a raw idea.
- Add tags.
- Mark whether it is for writing, product, research, or demo.
- Select one idea and generate a short outline.

Important boundary: do not build a social platform, sync service, or cloud account system in v0.1.

I want to use coding agents for the project, but I need every agent session to know the goal, current scope, and what not to build.
